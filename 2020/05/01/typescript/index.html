typescript 学习

1.基础

  • 全局安装：npm install typescript -g
  • 查看版本号：tsc -v
  • 编译：tsc greeter.ts
  • 初始化生成 tsconfig.json：tsc –init

2.基础类型

  • 布尔值：boolean
  • 数值：number
  • 字符串：string
  • 数组：number[]/Array
  • 元组：[string, number],已知元素数量和类型的数组
  • 枚举：enum Color {Red = 1, Green, Blue}，为数字赋名字，默认从 0 开始
  • any：任意类型的值
  • void：只能为它赋予 undefined 或 null，定义函数无返回值
  • null：对象值缺失，空对象的引用
  • undefined：初始化未定义
  • never：类型表示的是那些永不存在的值的类型
  • object：非原始类型
  • ｜：联合类型
  • 类型断言：可使用尖括号或 as，在 jsx 中只能使用 as

3.tsconfig.js 文件

compilerOptions：编译器选项

  • outDir：输出目录
  • allowJs：是否允许 js，jsx 文件被包含
  • target：编译后降级的版本
  • compileOnSave：是否让 IDE 在保存文件的时候重新生成文件
  • experimentalDecorators：是否启用实验阶段的装饰器功能
  • 编译选项

files：指定将被编译的文件，包含相对或绝对文件路径的列表
include：指定将被编译的文件夹或文件，一个文件 glob 匹配模式列表
exclude：指定不被编译的文件夹或文件，一个文件 glob 匹配模式列表

  • *：匹配 0 或多个字符
  • ？：匹配任意一个字符
  • **/：递归匹配任意子目录

4.增加的功能

  • 类型批注：编译时对数据的类型进行检查
  • 类型推断：根据上下文推断数据的类型
  • 接口：使用 interface 指定一个对象的属性及属性类型
  • 装饰器：一种特殊类型的声明，它能够非侵入式的被附加到类声明，方法， 访问符，属性或参数上
  • 装饰器的类型：类装饰器，访问器装饰器，属性装饰器，方法装饰器，参数装饰器
  • 范型：使函数可以接收任意类型的参数，并返回相同的类型
