sass 使用

1.优越体现

  • 可以定义变量，方便统一修改和维护
  • 可以进行选择器的嵌套，表示层级关系
  • 如导入其他 sass 文件，最后编译为一个 css 文件，优于纯 css 的@import
  • 可用 mixin 定义一些代码片段，且可传参数，方便日后根据需求调用
  • 可通过@extend 来实现代码组合声明，使代码更加简洁
  • 可进行简单的加减乘除运算
  • 集成了大量的颜色函数，让变换颜色更加简单

2.语法

  1. 文件名后缀
  • sass：不使用大括号和分号
  • scss：使用大括号和分号，建议使用
  1. 导入(可以忽略后缀名)
  1. 注释
  • //：单行注释
  • /* */：多行标准注释
  1. 变量
  • 普通变量
  • 默认变量：加上!default
  • 特殊变量：以#{$variables}形式使用
  • 多值变量：list，map
  • 全局变量：加上!global
  1. 嵌套：选择器嵌套，属性嵌套
  2. @at-root：跳出选择器嵌套
  3. 混合(mixin)：用@mixin 声明，通过@include 来调用
  1. 继承：使用关键词@extend
  2. 函数：以@fuction 开始
  3. 运算：注意运算符前后留一个空格
  4. 条件判断及循环
  • @if 判断
  • 三目判断
  • @for 循环
  • @each 循环

3.使用

  • 全局安装：npm i sass -g
  • 将 scss 文件转化为 css 文件，指定文件名时会自动生成并保存该文件
  • 四种编译风格选项
  1. nested：嵌套缩进的 css 代码，它是默认值
  2. expanded：没有缩进的、扩展的 css 代码
  3. compact：简洁格式的 css 代码
  4. compressed：压缩后的 css 代码
  • 监听文件或目录
  • 在线转换器
