vue-cli 2019/03/01

vue-cli 配置

1.设置启动项目时自动打开浏览器

2.设置 api 代理

3.给 img 文件夹配置别名

4.打包之后打开 index.html 页面空白

  • 当 vue-router 使用 history 的时候不用加点

5.打包之后 app.vue 中引入以及写入的 css 样式未生效

  • 不要在 app.vue 中写 css

6.打包之后 element-ui 图标资源加载失败

  • 添加 publicPath: ‘../../‘

7.打包之后 iconfont 在线引入报错

  • 下载 iconfont.js，在 main.js 中引入

8.打包之后显示静态页面无法或获取数据

  • 在 axios 中配置 api 前缀 baseURL

9.打包之后去掉 .map 文件

  • build 打包时间从 31s 减少到 27s

10.查看打包之后各文件的体积

  • npm run build –report

11.使用命名式路由懒加载

12.开启 gzip 压缩（8s-2s）

  • 安装依赖：cnpm install –save-dev compression-webpack-plugin@1.1.2(版本过高 build 时会报错)
  • 首屏从 8s 降低到 2s，性价比最高
  • 修改配置
  • npm run build 之后
  • 在 network 中点击 app.db335b540de0fe17b5dd953bf51481fa.css 或 vendor.9fd7869f1da6dd02362e.js 在响应头中找到 Content-Encoding: gzip 说明开启 Gzip 成功

13.图片压缩(报错)

  • 安装依赖：cnpm install image-webpack-loader –save-dev

14.加 title 前的小图标

  • 将图标放入 static 文件夹，在 index.html 内引入

15.性能分析 Hiper(失效)

  • 全局安装：npm install hiper -g

16.配置开发环境和生产环境不同 API 接口

  • 本地启动打印 development_api，线上环境打印 production_api

17.开启 CDN 加速（2s-0.5s）

  • 使用免费 CDN：bootcdn
  • css 影响很小可以忽略，不需要使用 CDN 加速

18.vue-cli3 的配置

  • 配置集中到了一个文件，页面加载速度变化不大
  • 如果页面中存在 console，默认在 build 打包的时候会报 eslint 错误

19.vue-cli3 多环境配置

  • 新建文件：.env.development，.env.qa，.env.staging，.env.production
  • 在 vue 或 js 文件中使用：process.env.VUE_APP_BaseUrl
  • 也可以通过判断域名设置不同的 api 前缀，并将其全局挂载到 Vue 的实例上
  • 只在用 npm run build 时，才会生成 css 文件，并开启 gzip
