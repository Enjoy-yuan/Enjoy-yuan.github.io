login 登录权限控制

1.登录一般流程

  • 在用户输入正确的用户名和密码后，后端将 token 及其它必要信息返回给前端
  • 前端使用 js-cookie 将其保存在 cookie 中
  • 需要权限的接口，将 token 塞入到请求头中，与后端约定 token 过期的返回值

2.不同二级域名可设置相同的顶级域名，使 token 共享

  • 正式环境与开发环境需要加以区分
  • 将 if 写到最外层会报错：’import’ and ‘export’ may only appear at the top level

3.设置路由拦截，当 token 不存在，则回到登录页

  • 给需要权限的路由添加 meta

4.token 过期的处理

  • 在请求拦截中将 token 塞入 header
  • 根据不同的域名设置不同的 baseURL，用来区分本地，测试，开发环境，或通过配置来区分
  • 保证有且仅有一次登录过期的弹框提示
  • 当存在扫码轮询时，进度条动画会比较频繁，可将其关闭

5.在 nuxt 项目中的注意点

  • 路由的跳转需要先判断当前环境，router 不再可用
