  • 无法使用@img

6.vant 移动端适配

  • 内联样式不会自动转 rem
  • rootValue 或 viewportWidth 的值不建议修改，否则 vant 组件的大小也会随之变化

7.keep-alive 的使用

  • 定义 两个 vue 文件，使用 keep-alive 包裹它们
  • 在两个路由之间切换，查看生命周期的运行情况
  • 路由第一次进入时会被缓存，再次进入时生命周期函数不会被执行
  • 合理利用 keep-alive 可以减少页面请求

8.refs 与使用原生方法获取 dom 元素的区别

  • ref 获取的是虚拟的 dom，可以有效减少性能消耗
  • 通过 ref 可以直接调用子组件内部的方法或数据

9.执行过程

  • 使用 new Vue 初始化，合并配置，初始化生命周期，初始化事件中心，初始化渲染，初始化 data，props，computed，watcher 等
  • Vue 是用 function 实现的类，new Vue 的时候生成实例化对象，拥有 Vue 原型上的属性和方法，并将 this 指针指向该实例化对象
  • 通过$mount 将实例去挂载到 dom 元素上，再通过 render 函数生成虚拟 dom 树 VNode
  • 渲染过程中的数据会通过 getter 被 Watcher 收集，若数据发生变化，会通知 Watcher 执行回调函数，再通过 updata 来把 VNode 渲染成真实的 dom
  • 若存在 template，会将 template 编译成 render 方法，若存在 jsx，同样会被编译成 render 方法
  • 在 render 方法内部通过返回 createElement 来创建元素生成虚拟 dom 树，通过 VNode 这个 class 去定义

9.响应式原理

  • proxy 可以监听整个对象
  • 通过 Object.defineProperty 给实例对象的所有属性添加 setter 和 getter 方法，getter 来进行依赖收集，setter 进行通知更新
  • 在组件的渲染过程中，通过 watcher 进行依赖的收集，当依赖项的 setter 触发时，会通知 watcher，然后将 watcher 添加到队列中，在 nextTick 之后将关联的组件进行更新
  • 弊端：数组在利用索引来修改值，或者修改长度的时候不会被监听，对象不存在的属性无法被监听，当对象嵌套的层级比较深时，递归遍历性能不好

10.vuex 状态管理

  • 取值：this.$store.state/getters/commit/dispatch
  • 辅助函数：mapState，mapGetters，mapMutations，mapActions
  • 相比全局变量来说，vuex 中的 state 是响应式的，且可以保存异步数据，单向数据流
  • mutations 中同样可以执行异步操作，使用 actions 便于使用 devtools 调试时，更好的追踪数据的状态变化
  • pinia：组合式风格的 API，更好的 TS 支持，弃用 mutations，使用 setup 无需使用辅助函数

11.vue3 相比 vue2 的优化

  • 组合式风格的 API
  • 更好的 TS 支持
  • 响应式变化改用 Proxy
  • 优化了虚拟 DOM 的 diff 算法
  • 默认使用了 Tree Shaking
  • 使用高阶组件而不是构造函数来初始化 app
  • 使用 hooks 可以更好的状态复用
  • 允许返回多个根结点
  • 使用更小更快的 pinia 替换 vuex

12.vue 与 react 的不同

  • vue 使用双向数据绑定，react 使用单向数据流
  • vue 的 api 和约束更多，react 使用 JSX

13.qiankun 微前端

  • router-view 外不得使用 transition 包裹，会导致 dom 未加载的报错
  • 微前端的子应用二级域名需要在 nginx 中设置域名允许被跨域

15.常用修饰符

  • .stop：阻止冒泡，从子组件到父组件，默认
  • .capture：捕获阶段触发，从父组件到子组件
  • .prevent：阻止默认行为
  • .self：目标自身触发
  • @keyup.enter：回车键触发
  • .middle：鼠标中间键触发
  • v-model.number：只允许输入数字
  • v-model.trim：去除两端空格
